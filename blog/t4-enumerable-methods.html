#map

08/01/2015

Mapping out the #map

During this week, we were tasked with exploring ruby-docs in order to learn/try methods in order to solve problems. Although I didn't end up using #map this week for any of my problems, I was able to figure out that it was a very important ruby method that Rubyists seem to prefer using in their code. As a result, it behooves us to explore this method and figure out why it is so damn preferable.

What is it?

Ruby-docs says “the map method can be used to create a new array based on the original array, but with the values modified by the supplied block.” Okay.

So WHAT DOES IT MEAN?

Well, I guess it would be best to compare it to a much more familiar process which is the each method we are accustomed to. Recall that when we call #each, we are iterating over each element in an array or a hash. Well, #map essentially does the same thing as well. 

arr.map { |a| 2*a }
arr.each {|a| 2*a}

However, there is a big difference between #each and #map. Remember that each is not producing a new object or even affecting the original object. As a result, you could tell it to print the result of the code as it runs through it, but it ultimately returns the original array or hash that it started out with. Of course, you could create a new object by setting a variable to this method, but programmers prefer the easier, simpler functionality. Why do extra steps when you could get a very similar method that could also create an object?

#map will iterate over your items like #each, but it also returns a new object albeit an array or a hash that reflects the code block applied Instead of returning the original item it worked off, it will return a new product/form. This is useful for cutting down on pseudocode obviously and making more efficient code that is supposedly easier to read and debug. So consider our example from earlier. 

arr= [1,3,5]
arr.map { |a| 2*a } ==> [2,6,10] 
arr.each {|a| 2*a} ==>  [1,3,5]

As you can see, one returns the original while the other returns the result of the code block. Remember , we can incorporate arr. each in a way so that it can return the same results. For example: 


def twice_ply(number)
  new = Array.new
  number.each do | element |
    new << 2*element
  end
  return new
end

However, the map version would probably be simpler than this.

def twice_ply (number)
number.map{| element |  2 *element |
end

As you can see, one is way easier to read simply due to cutting down some steps.
