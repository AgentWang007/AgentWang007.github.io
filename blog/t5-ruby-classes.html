Classes

Toy Factory Analogy

08/05/2015

Intro

Classes are blueprints. Imagine if classes were blueprints being made by a toy factory. Now, imagine if you own this factory and it is christmas time. Do you want the workers to basically create a new recipe in order to create slightly different variations on an action figure or a pony? NOPE. It’s not practical, right? It would make sense to have your workers to work off blueprint with some modifiable methods, right? This is class in a nutshell. It’s a giant blueprint to make objects!

So how do we go about making a class? Well, it sort of looks like this: 


class Figures

end

All classes start with a capital letter. In between the beginning and the end, you can throw in methods that you expect each object to perform. You can sort of think of it as making sure that all your action figures have pose-able limbs or shoot plastic projectiles. Within every class, you have to create a method called initialize . This is a necessary method that all you to make a new instance(object or toy) of the class! Therefore: 


class Figures
def initialize (name)
@name=name
end
end

p Figures.new(‘Army_Joe')=>
#

There is our action figure (as a symbol)! However, it’s doesn’t do anything. How about we make it do something else? How about a talking function? We could call the method “speak.” It would make every instance of this class able to say a funny one-liner. 


class Figures
def initialize (name)
@name=name
end
  def speak
  puts "hello #{@name}!"
end
end

Joe=Figures.new("Joe")
p Joe.speak = > “hello Joe!”

As you can see, we created a product that says a phrase. We were able to do this by creating a method (which you can think of as a feature product on the toy) in the class (our blueprint). Now, you are probably staring at the @ in front of our variables and wondering what it is. @ are used to create special variables that can be used within the class. Remember that local variables work within their environment, but can’t be applied outside of it. However, instance variables can be seen from within the class and by any instance of the class. You can think of it as each object getting their own instance of the variable created in the class. Therefore, in our previous example, the instance (the figure we are making) gets his own copy of the variable (“Joe”).
